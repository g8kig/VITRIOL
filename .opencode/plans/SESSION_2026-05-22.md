## Context Size vs Pin Pool Tradeoffs

On GTX 1070 Ti (8 GB VRAM) with Qwen3.6-35B-A3B-UD-IQ2_M + MTP N=2:

| Context | Pin Layers | KV Cache | Pin Pool | Gen Speed | Notes |
|---------|-----------|----------|----------|-----------|-------|
| 8,192 | 8 | 28 MiB | 1.6 GiB | **25.2 tok/s** | Default, max throughput |
| 136,192 | 4 | 1.7 GiB | 792 MiB | **21.5 tok/s** | Explicit `--pin-layers 4` |
| 262,144 | 0 (auto) | 3.3 GiB | 0 | **21.1 tok/s** | Auto-disabled >64K |

### Auto-Disable Logic

The `vitriol` script auto-disables pin pool at context > 65536 to prevent OOM.
The KV cache at 262K takes 3.3 GiB â€” without auto-disable, the pin pool (1.6 GiB)
plus MTP head KV (328 MiB) would exceed 8 GB VRAM.

**CLI override:** `--pin-layers N` bypasses auto-disable (explicit user intent).
